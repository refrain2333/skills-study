### 集成你的 API

1. 打开 `研究助手系统示例.py`
2. 找到 `LLMInterface` 类的 `call()` 方法
3. 参考 `LLM-API集成指南.md` 选择你的提供商
4. 替换实现
5. 运行测试

**示例：集成 OpenAI**

```python
def call(self, prompt, system_prompt=None, temperature=0.7, max_tokens=2000):
    from openai import OpenAI
    client = OpenAI(api_key=self.api_key)
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": prompt}
        ]
    )
    return response.choices[0].message.content
```

更多选项见 `LLM-API集成指南.md`。

## 💡 核心原理（速查）

### 原理 1：上下文是有限资源
```
❌ 更大的上下文 = 更好
✅ 精心选择的信息 = 更好

质量 > 数量
```

### 原理 2：多代理架构
```
单代理（容易超载）
    ↓
多代理（分工协作）
    ├─ 搜索代理
    ├─ 分析代理
    └─ 综合代理

每个代理有清晰的上下文 = 效果更好
```

### 原理 3：工具设计
```
❌ 15 个特定工具（代理困惑）
✅ 3 个综合工具（代理清晰）

原则：如果你自己不知道用哪个工具，代理也不知道
```
