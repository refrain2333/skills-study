## 🔑 核心要点总结

### 1️⃣ 上下文是有限资源
```
❌ 更大的上下文 = 更好（这是错误的！）
✅ 精心选择的信息 = 更好（正确的）

质量 > 数量
```

### 2️⃣ 多代理架构的优势
```
单代理：一个代理处理所有（容易超载）
多代理：多个代理分工（各自专注，效果更好）
```

### 3️⃣ 工具设计决定效果
```
❌ 15 个特定工具 → 代理困惑
✅ 3 个综合工具 → 代理清晰
```

### 4️⃣ 管道模式是标准做法
```
Acquire → Prepare → Process → Parse → Render
```

### 5️⃣ 文件系统是状态管理
```
data/batch_001/item_001/
├── raw.json      (获取完成)
├── prompt.md     (准备完成)
├── response.md   (处理完成)
└── parsed.json   (解析完成)
```
